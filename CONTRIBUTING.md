# Contributing to LazyOwnInfiniteStorage

Thank you for your interest in contributing to LazyOwnInfiniteStorage! Below you will find the guidelines for contributing to the project.

## How to Contribute

### Reporting Issues

If you find a bug or have a suggestion to improve the project, please open an issue on GitHub. Include a detailed description of the problem or suggestion, and provide all the necessary information so we can reproduce the issue or better understand the suggestion.

### Pull Requests

Contributions are welcome in the form of pull requests. Here are some guidelines to ensure your contribution can be integrated smoothly:

1. **Fork the repository** and clone your fork locally.

    ```sh
    git clone https://github.com/your-username/LazyOwnInfiniteStorage.git
    cd LazyOwnInfiniteStorage
    ```

2. **Create a branch for your contribution**. Use a descriptive name for your branch.

    ```sh
    git checkout -b your-branch-name
    ```

3. **Make your changes**. Make sure your code follows the project style and that you have added the necessary tests.

4. **Commit your changes**. Make sure to write a clear and descriptive commit message.

    ```sh
    git commit -m "Clear description of the changes"
    ```

5. **Push your changes to your fork**.

    ```sh
    git push origin your-branch-name
    ```

6. **Open a pull request**. Go to the original repository on GitHub and open a pull request from your fork. Describe the changes you have made and why they are useful.

### Code Style

Keep a coding style consistent with the rest of the project. This includes:

- Following the project's naming and style conventions.
- Making sure your code is properly indented and formatted.
- Adding comments where necessary to explain your code logic.

### Testing

Make sure your code has been properly tested. If you add new features, you should also add tests for them. If you fix a bug, add a test that demonstrates the fix.

## Communication

If you have any questions or need to discuss something, open an issue on GitHub or contact via Discord. We are here to help you and collaborate.
